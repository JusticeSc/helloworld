# F1 Score 深度解构：从调和平均的本质到业务落地的终极指南

---

## 一、第一性原理：为什么必须是调和平均？

### 1.1 直觉陷阱：算术平均的谎言

很多人的第一反应是：

$$\text{Arithmetic Mean} = \frac{P + R}{2}$$

直观，对称，且——在比率型指标上有严重误导性。

看一个极端例子：

| 模型             | Precision | Recall | 算术平均  | F1 (调和平均) |
| :--------------- | :-------: | :----: | :-------: | :-----------: |
| 模型 A（全猜正） |   0.01    |  1.00  | **0.505** |     0.020     |
| 模型 B（保守）   |   1.00    |  0.01  | **0.505** |     0.020     |
| 模型 C（平衡）   |   0.50    |  0.50  |   0.500   |   **0.500**   |

算术平均给模型 A 和 B 打了 **0.505 分**——甚至超过了真正平衡的模型 C。这是荒谬的：一个几乎全错（P=0.01）或几乎全漏（R=0.01）的模型，凭什么及格？

### 1.2 数学本质：调和平均的"短板惩罚"

F1 Score 的定义是 Precision 和 Recall 的**调和平均数**：

$$F1 = \frac{2 \cdot P \cdot R}{P + R}$$

或等价地：

$$\frac{1}{F1} = \frac{1}{2}\left(\frac{1}{P} + \frac{1}{R}\right)$$

**直觉锚点——往返速度问题**：你开车从 A 到 B，去程 60 km/h，回程 40 km/h。全程平均速度是多少？

- 直觉误答：50 km/h（算术平均）
- 正确答案：**48 km/h**（调和平均）

因为去程耗时 1 小时，回程耗时 1.5 小时——你在慢速路段**消耗了更多时间**，慢速拖了后腿。调和平均正是对"单位产出"（每公里时间）取平均后再取倒数，天然惩罚短板。

同理，在分类任务中：
- 若 $R \to 0$（大量漏报），无论 P 多高，F1 $\to 0$
- 若 $P \to 0$（大量误报），无论 R 多高，F1 $\to 0$

> **核心性质**：$\min(P, R) \leq F1 \leq \max(P, R)$，且 F1 总是比算术平均更靠近较小值。只要 P 或 R 中有一个为 0，F1 即为 0。

## 二、三重透视：用三种语言理解同一个 F1

### 2.1 集合论语言：Dice 系数

设 $A$ = 模型预测为正的样本集合，$B$ = 真实正样本集合：

$$F1 = \frac{2|A \cap B|}{|A| + |B|} = \frac{2 \cdot TP}{2TP + FP + FN}$$

```
        ┌──────────────────────────┐
        │        全集 U            │
        │   ┌──────┐    ┌──────┐   │
        │   │  A   │ TP │  B   │   │
        │   │ (FP) │ ∩  │ (FN) │   │
        │   │      │    │      │   │
        │   └──────┘    └──────┘   │
        │                         │
        │        TN (无关)         │
        └──────────────────────────┘
```

三条洞察：
- **TN 完全被忽略**——F1 只关心"你说对了吗"和"你漏了吗"，不关心"你正确忽略的东西"。
- **分子是交集**，分母是并集的扩展——两个集合越重合、越紧凑，F1 越高。
- **F1 是 Jaccard 系数的单调变换**：$F1 = \frac{2J}{1+J}$，其中 $J = \frac{TP}{TP+FP+FN}$。

### 2.2 成本会计语言：单位收益的错误代价

将 F1 的倒数展开：

$$\frac{1}{F1} = \frac{TP + FP + FN + TP}{2TP} = 1 + \frac{FP + FN}{2 \cdot TP}$$

重新排列：

$$F1 = \frac{TP}{TP + \frac{FP + FN}{2}}$$

**翻译成人话**：
- 分母 = 正确预测数 + 平均错误数
- F1 = 正确预测占"总代价"的比例
- **每获得 1 个 TP，你需要额外承担 $\frac{FP+FN}{2TP}$ 个"平均错误"**

数值示例：

| TP   | FP   | FN   | F1    | 解读                                           |
| :--- | :--- | :--- | :---- | :--------------------------------------------- |
| 100  | 10   | 0    | 0.952 | 每 1 个正确，仅附带 0.05 个错误                |
| 100  | 50   | 50   | 0.667 | 每 1 个正确，附带 0.5 个错误                   |
| 100  | 100  | 100  | 0.500 | 每 1 个正确，附带 1 个错误——模型几乎没有区分力 |

### 2.3 几何语言：偏科的代价函数

由均值不等式 $AM \geq GM \geq HM$：

$$\frac{P+R}{2} \geq \sqrt{PR} \geq \frac{2PR}{P+R}$$

当 $P$ 和 $R$ 差值越大，调和平均**向下偏离算术平均越远**：

| P    | R    | 算术平均 | F1       | 偏差 |
| :--- | :--- | :------- | :------- | :--- |
| 0.9  | 0.9  | 0.90     | 0.90     | 0%   |
| 0.9  | 0.5  | 0.70     | **0.64** | 8.6% |
| 0.9  | 0.1  | 0.50     | **0.18** | 64%  |
| 1.0  | 0.0  | 0.50     | **0.00** | 100% |

F1 在 P 和 R 之间构建了一个**凹性惩罚曲面**——偏科的成本是非线性的，越偏越痛。

## 三、$F_\beta$：当"平等"不成立时

F1 隐含假设：FP 和 FN 的代价相等。现实几乎从不如此。

### 3.1 通用公式

$$F_\beta = (1+\beta^2) \cdot \frac{P \cdot R}{(\beta^2 \cdot P) + R}$$

其中 $\beta$ = Recall 相对于 Precision 的重要性倍数：

- $\beta = 1$：同等重要 → F1
- $\beta > 1$：Recall 更重要 → 更容忍 FP
- $\beta < 1$：Precision 更重要 → 更容忍 FN

### 3.2 常用取值与业务映射

|  $\beta$   | 名称 | 侧重              | 典型场景                                 | 业务逻辑                           |
| :--------: | :--- | :---------------- | :--------------------------------------- | :--------------------------------- |
|    0.5     | F0.5 | Precision 权重 2× | **垃圾邮件过滤**、**内容审核**           | 误删一封正常邮件比漏拦十封垃圾更糟 |
|    1.0     | F1   | 平衡              | **通用基准**、**信息抽取**               | 无先验偏好                         |
|    2.0     | F2   | Recall 权重 2×    | **癌症筛查**、**逃犯识别**、**地震预警** | 漏掉一个比多检查十个代价大得多     |
| $\sqrt{2}$ | —    | 自定义            | **欺诈检测初筛**                         | 宁可多派人工审核，不能放过可疑交易 |

### 3.3 极限行为

- $\beta \to 0$：$F_\beta \to P$（只看 Precision）
- $\beta \to \infty$：$F_\beta \to R$（只看 Recall）
- 两者之间：平滑地调节 FP 与 FN 的相对代价

> **操作原则**：不要凭感觉选 $\beta$。找业务方问清楚——"漏一个坏样本的损失，等于误拦几个好样本？" 让业务给出比值，然后设 $\beta = \sqrt{\text{cost}(FN) / \text{cost}(FP)}$。

## 四、决策框架：何时用 F1，何时果断不用

这是整篇文章**ROI 最高**的部分。选错指标比选错模型更致命——它直接让优化方向跑偏。

### 4.1 决策树

```
你在评估分类模型
  │
  ├─ 类别是否严重不平衡（正例 < 10%）？
  │    ├─ 是 → 进入不平衡分支
  │    │    ├─ FP 和 FN 代价大致相等？ → F1 ✓
  │    │    ├─ FN 代价明显更大？ → F2 或 PR-AUC ✓
  │    │    └─ FP 代价明显更大？ → F0.5 或 Specificity ✓
  │    │
  │    └─ 否 → 类别平衡
  │         ├─ 需要评估排序或概率质量？ → AUC-ROC / Log Loss ✓
  │         ├─ 只需整体准确率？ → Accuracy ✓
  │         └─ 需要综合指标？ → MCC (Matthews) ✓
  │
  └─ 多分类？
       ├─ 在乎每个类的表现 → Macro-F1 ✓
       ├─ 在乎整体正确率 → Micro-F1 (=Accuracy) ✓
       └─ 折中 → Weighted-F1 ✓
```

### 4.2 适用场景深度解析

**✅ 类别极度不平衡**

Accuracy 在不平衡数据上的崩溃是最经典的评估事故：

| 真实分布          | 模型策略 | Accuracy |
| :---------------- | :------- | :------- |
| 正例 1%, 负例 99% | 永远猜负 | **0.99** |

一个什么也不学的模型拿到了 99% 的准确率——Accuracy 在此完全失灵。F1 忽略 TN，直接聚焦少数类表现，不会被多数类"淹没"。

**✅ "既要又要"的场景**

搜索引擎——既不能返回大量无关结果（P↓），也不能漏掉关键页面（R↓）。推荐系统同理。

**✅ Macro-F1：保护长尾类别**

Micro-F1 按样本量加权，大类主导；Macro-F1 按类别平均，每个类一票。

当你的电商平台有 1000 万手机 SKU 和 100 个古董配件 SKU，Micro-F1 可以完全忽略配件分类的糟糕表现而仍然光鲜亮丽。Macro-F1 会揭穿这一点。

### 4.3 不适用场景深度解析

**❌ 数据平衡且两类同等重要**

TN 包含有效信息。F1 丢弃它，意味着你损失了数据中一半的分布信息。此时应使用 **MCC (Matthews Correlation Coefficient)** 或 **AUC-ROC**。

MCC 的特殊价值：

$$\text{MCC} = \frac{TP \times TN - FP \times FN}{\sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)}}$$

MCC 是唯一同时利用混淆矩阵四个象限的综合指标，取值范围 [-1, +1]，0 表示随机猜测。在平衡数据上，MCC 比 F1 信息量更完整。

**❌ 代价极度不对称**

如果漏报一个欺诈交易损失 10 万元，误报一个正常交易只需 0.1 元人工审核费，代价比 = 100,000:1。此时用 F1（比 1:1）是对业务信号的严重扭曲。请使用 **期望损失（Expected Cost）** 作为优化目标：

$$\text{Cost} = FP \cdot C_{\text{FP}} + FN \cdot C_{\text{FN}}$$

**❌ 评估排序质量或概率校准**

F1 依赖单一阈值。两个模型可能：
- 模型 M：输出概率在 0.48~0.52，随机波动
- 模型 N：输出概率在 0.01~0.99，置信分明

在阈值 0.5 下，两者 F1 可能相同——但 N 明显更好。此时用 **PR-AUC**（不平衡数据）或 **ROC-AUC**（平衡数据）更合理。

**❌ 关注模型的整体区分潜力**

F1 只是 PR 曲线上的**一个点**。换个阈值，F1 可能大变。PR-AUC 则覆盖了所有阈值下的综合表现：

```
PR 曲线示意：
P ↑
  │  ●  ← 高阈值：高 P 低 R
  │     ●
  │        ●  ← 你的 F1 只是这条曲线上某个点
  │           ●
  │              ●  ← 低阈值：低 P 高 R
  └──────────────────→ R
```

## 五、多分类 F1：Micro / Macro / Weighted 的精确含义

这是工程实践中**最容易混淆**的地方。

### 5.1 三种聚合策略

设 $TP_i, FP_i, FN_i$ 为第 $i$ 类的计数，共 $K$ 个类别。

| 策略         | 公式                                                 | 本质             | 使用时机              |
| :----------- | :--------------------------------------------------- | :--------------- | :-------------------- |
| **Micro**    | 先全局求和 $TP, FP, FN$，再算 F1 $\equiv$ Accuracy   | 每个样本权重相同 | 关心整体吞吐量        |
| **Macro**    | $\frac{1}{K}\sum_i F1_i$                             | 每个类别权重相同 | 关心尾部类别公平性    |
| **Weighted** | $\sum_i w_i \cdot F1_i$，$w_i$ = 类别 $i$ 的样本占比 | 类别权重按样本量 | Macro 和 Micro 的折中 |

### 5.2 关键陷阱

**Micro-F1 = Accuracy**（在多分类中恒等式成立）：

$$\text{Micro-F1} = \frac{2\sum TP_i}{2\sum TP_i + \sum FP_i + \sum FN_i} = \frac{\sum TP_i}{N} = \text{Accuracy}$$

因为 $\sum FP_i = \sum FN_i$（每个预测错误在某个类上是 FP，在另一个类上是 FN）。

**实践建议**：报告多分类结果时，同时给出 Macro-F1 和 Weighted-F1。两者差值越大，说明小类表现越差。

## 六、实战清单：从理论到代码

### 6.1 务必报告的配套指标

单报 F1 是危险的。完整报告应包含：

```
必报组合：
├── F1 Score（或 Fβ）
├── 混淆矩阵（原始计数）
├── Precision & Recall（分别报告）
├── PR-AUC（不平衡数据）或 ROC-AUC（平衡数据）
└── 目标类的 Support（样本数）
```

### 6.2 阈值校准的标准流程

```python
import numpy as np
from sklearn.metrics import f1_score, precision_recall_curve

def find_optimal_threshold(y_true, y_proba, beta=1.0):
    """
    在验证集上搜索使 F_beta 最大化的阈值。
    严禁在测试集上执行此操作。
    """
    precision, recall, thresholds = precision_recall_curve(y_true, y_proba)
  
    # F_beta 计算（向量化）
    beta_sq = beta ** 2
    f_scores = (1 + beta_sq) * (precision * recall) / (beta_sq * precision + recall + 1e-12)
  
    # 去除 PR 曲线的最后一个点 (R=0, P=1，无对应阈值)
    f_scores = f_scores[:-1]
  
    best_idx = np.argmax(f_scores)
    return thresholds[best_idx], f_scores[best_idx]


# 使用示例
# ① 拆分：train / val / test（阈值搜索只在 val 上）
# ② 在 val 上搜索最佳阈值
best_thresh, best_f2 = find_optimal_threshold(y_val, model.predict_proba(X_val)[:, 1], beta=2.0)
# ③ 用找到的阈值在 test 上评估——只评估一次
y_pred_test = (model.predict_proba(X_test)[:, 1] >= best_thresh).astype(int)
final_f2 = f1_score(y_test, y_pred_test)  # 注意：sklearn 的 f1_score 可通过 beta 参数支持 F-beta
```

### 6.3 警惕"阈值黑客"

一种隐蔽的过拟合形式：在测试集上反复调整阈值直到 F1 漂亮，然后报告这个数字。这是**数据泄露**——阈值本身就是模型的一个超参数。

正确流程：
```
训练集 → 训练模型
验证集 → 选择阈值 + 选择超参
测试集 → 仅评估一次，使用验证集确定的阈值
```

### 6.4 sklearn 中的常见坑

```python
from sklearn.metrics import f1_score

# ⚠️ 默认 average='binary'——多分类会报错
f1_score(y_true, y_pred)  # 仅适用于二分类

# ✅ 多分类显式指定
f1_score(y_true, y_pred, average='macro')   # 每个类平均
f1_score(y_true, y_pred, average='weighted') # 样本量加权
f1_score(y_true, y_pred, average=None)       # 返回每个类的 F1 列表

# ✅ F-beta 通过 beta 参数
f1_score(y_true, y_pred, beta=2.0, average='macro')  # Macro-F2
```

## 七、历史注记与延伸阅读

F-measure 的起源可追溯到 **Cornelis Joost van Rijsbergen** 在 1979 年出版的《Information Retrieval》一书。他定义了 Effectiveness ($E$) 度量：

$$E = 1 - \frac{1}{\alpha \cdot \frac{1}{P} + (1-\alpha) \cdot \frac{1}{R}}$$

其中 $\alpha = \frac{1}{1+\beta^2}$。当 $\alpha = 0.5$（即 $\beta = 1$），$E$ 的补数就是今天的 F1。

这并不是巧合——信息检索天然关心"找到的文档有多少相关"（Precision）和"相关文档找到了多少"（Recall），而且两者同样重要。F1 从诞生之初就被刻入了"平衡"的基因。

## 八、结语：指标即价值观

回望整篇文章，我们穿越了：

- **调和平均的数学必然性**——它惩罚偏科，且惩罚是非线性的
- **三重透视的直觉**——集合重叠、单位成本、几何偏差
- **$F_\beta$ 的自由度**——当 FP ≠ FN 时，你有权选择在乎谁多一些
- **决策框架**——不是所有场景都欢迎 F1，有时它甚至是错误的选择
- **实战纪律**——阈值分离、多分类策略、配套报告

最终的认知不是"F1 好不好"，而是：

> **每一个评估指标，都是对业务价值观的数学编码。**

F1 编码了三件事：
- 我在乎少数类（忽略 TN）
- 我不容忍偏科（调和平均）
- 我认为 FP 和 FN 代价相等（$\beta = 1$）

当这三个前提任意一个不成立时，请诚实地换一个指标。

**没有最好的指标，只有最懂业务的指标。**

下次写下 `f1_score(y_true, y_pred)` 时，问自己三个问题：

1. **我的正负例平衡吗？** 如果平衡，Accuracy 或 MCC 可能更好。
2. **FP 和 FN 的代价真的相等吗？** 如果不相等，调 $\beta$。
3. **我真的只关心一个阈值点吗？** 如果关心整体潜力，看 PR-AUC。

这三个问题的时间成本不到 30 秒，但它能确保你的优化方向不偏离业务目标。这才是数据科学家的专业素养所在——**不是会算指标，而是知道该算哪个指标。**
